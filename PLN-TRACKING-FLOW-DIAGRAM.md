## ğŸ¯ Key Points

### Universal PIN
- **All applications** use PIN `"12345"`
- **No exceptions** - this is hardcoded
- **Backend validates** PIN before database query

### Reference ID
- **Format:** `PLN-YYYY-XXXXXXXXXXXX`
- **Case-sensitive:** Must be uppercase
- **Unique:** Each application has unique ID
- **Generated:** Automatically on creation

### Database Query
- **After PIN validation:** Only queries if PIN is correct
- **Searches by:** `referenceId` field only
- **Returns:** Full application document if found

### Error Messages
- **Clear and specific:** Tells user exactly what's wrong
- **Actionable:** Suggests what to do next
- **User-friendly:** Non-technical language
